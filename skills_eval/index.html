Headline findings

The "When to Activate" duplication

In every group, ~7 of 10 skills carry a "When to Activate" or "When to Use" body section that mirrors the description's triggers. For a model-invoked skill the description is the single source of truth for invocation; the body section is pure context load.

Sprawl without disclosure

Eight skills exceed 400 lines with zero progressive disclosure. git-workflow (716), data-scraper-agent (766), python-testing (818), python-patterns (752), frontend-patterns (658), motion-ui (576), react-performance (576), dart-flutter-patterns (565). The fix is the same in every case: push branch-specific reference behind context pointers.

Leading-word gap

The top-quartile skills all anchor on pretrained concepts: caveman, tight, tracer bullet, PICO, garbage collection, throwaway, baseline/increment/trend. The bottom quartile relies on structural organisation or coined domain terms that recruit no priors.

Description-as-index, not trigger

Descriptions in git-workflow, frontend-patterns, flutter-dart-code-review, dart-flutter-patterns, and the homelab cluster enumerate content categories instead of trigger phrasing. They fail to fire reliably because they tell the model what's in the skill, not when to reach for it.

Missing completion criteria

Workflow skills frequently end without a checkable bar (brand-voice, grill-with-docs, lead-intelligence, deep-research Step 5, product-lens). This invites premature completion — the agent declares done before the work is actually exhaustive.

Template sediment

The three Python-family reference skills (python-patterns, python-testing, pytorch-patterns) all share the same defect: the description's first sentence is duplicated verbatim as the body's opening. Likely a shared template; fix in one pass.
